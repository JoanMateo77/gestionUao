**Actividad Asíncrona: Justificación de la arquitectura del sistema, backend, frontend y tecnologías** 

Como parte del desarrollo del proyecto, cada equipo deberá identificar y justificar de manera argumentada: 

- La arquitectura global del sistema, considerando el modelo de comunicación y la forma de despliegue, 
- la arquitectura del backend, 
- la arquitectura o patrón de diseño del frontend, 
- y la selección de tecnologías y frameworks para su implementación. 

La justificación debe evidenciar coherencia entre los requisitos del proyecto y las decisiones técnicas adoptadas, considerando atributos de calidad como mantenibilidad, escalabilidad, rendimiento y claridad estructural. Por lo tanto, es crucial que esté respaldada por un análisis  exhaustivo  y  argumentos  sólidos  a  través  referencias  bibliográficas  en  un documento escrito, el cual deberá estar conformado entre dos (2) y cuatro (4) páginas.  

**Consideraciones Generales** 

- La arquitectura seleccionada debe estar alineada con las características del sistema, como su tamaño, complejidad, escalabilidad y mantenibilidad. 
- Se debe incluir un cuadro comparativo de al menos dos alternativas arquitectónicas, indicando ventajas, desventajas y razones de la elección final. 
- Es importante diferenciar entre: 
  - arquitectura del sistema (cliente-servidor, monolito), 
  - arquitectura del backend (por capas), 
  - arquitectura del frontend (basada en componentes u otro patrón). 
- Se deben especificar las tecnologías y frameworks que se utilizarán para el desarrollo del frontend, backend y base de datos. 
- Las  decisiones  deben  estar  sustentadas  con  referencias  bibliográficas  o  técnicas confiables. 
- Evitar confundir tecnologías con arquitectura (ejemplo: React o Spring Boot no son arquitecturas). 

**Contenido del documento** 

El documento debe incluir: 

1. **Descripción general del sistema** 
- breve explicación del sistema a desarrollar 
- funcionalidades principales 
- contexto de uso 
2. **Selección y justificación de la arquitectura** Debe incluir: 
- **Arquitectura global del sistema** 
- Identificar el modelo de comunicación (cliente-servidor) 
- Identificar el modelo de despliegue (monolito) 
- Explicar la organización en capas (presentación, lógica, datos) 
- **Arquitectura del backend** 
- Describir la estructura interna (por capas) 
- Ejemplo: Controller – Service – Repository 
- Justificar la separación de responsabilidades 
- **Arquitectura del frontend** 
- Explicar cómo se organiza el frontend 
- Enfoque basado en componentes 
- Separación de: 
- **vistas** 
- **componentes reutilizables** 
- **servicios de consumo de API** 
- **Cuadro comparativo de arquitecturas** 
- mínimo dos alternativas 
- ventajas 
- desventajas 
- justificación de la elección 
3. **Selección de tecnologías y frameworks** 
- frontend (React, Angular, etc.) 
- backend (Spring Boot, etc.) 
- base de datos (MySQL, etc.) 

Cada elección debe justificarse en función del proyecto. 

4. **Referencias** 

fuentes bibliográficas o técnicas confiables. Usar una herramienta para su creación.
